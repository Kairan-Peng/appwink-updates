AppWink 0.1.14 Release Notes

AppWink v0.1.14

The first Sparkle-enabled release candidate for AppWink. This build keeps the existing app behavior while adding the metadata and distribution plumbing needed for in-app update checks from a public feed.

What's New

  • Signed release builds now embed a real SUFeedURL and SUPublicEDKey, so AppWink can participate in a public Sparkle update channel instead of relying only on manual download installs.
  • Toolbox and the menu bar now both expose Check for Updates…, and Toolbox also exposes the persisted automatic-update-check preference.
  • Release tooling now exports HTML release notes and can publish a signed appcast.xml plus hosted archives into the separate appwink-updates public repository.

Install Notes

  • Keep AppWink.app installed at /Applications/AppWink.app if you want the cleanest in-place upgrade path.
  • This release is intended to replace v0.1.13 (14) and should be the first candidate used for real public updater verification.

Known Limitations

  • The updater feed is only fully valid once this Sparkle-enabled signed artifact has been published; the older v0.1.13 DMG cannot be retrofitted into the first signed appcast item.
  • Fully automated verification of AppWink's Carbon global hotkeys is still unreliable in this development environment, so final hotkey closure still depends on real manual smoke.

Verification Scope

  • swift build --disable-sandbox and swift test --disable-sandbox completed successfully after integrating the updater runtime and release tooling.
  • Local loopback updater smoke succeeded with a temporary http://127.0.0.1:8000/appcast.xml feed serving both appcast.xml and the packaged archive.
  • The public updates host skeleton is live at https://kairan-peng.github.io/appwink-updates/; this release is the first candidate intended to populate that host with a real signed appcast item.
