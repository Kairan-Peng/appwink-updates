AppWink 0.1.15 Release Notes

AppWink v0.1.15

This release turns the first Sparkle-enabled feed cut into the next public GitHub release and closes the remaining launch-at-login usability gap on signed builds.

What's New

  • Toolbox and menu bar update surfaces remain backed by the live Sparkle feed, so packaged builds can check for updates from the hosted appcast.xml.
  • Launch at Login now lives in Toolbox > Other, keeping the hotkey home focused on bindings and other app-level settings grouped together.
  • Signed builds no longer leave the Launch at Login row permanently disabled before first registration: a pre-registration SMAppService.mainApp.status == .notFound state is now treated as retryable, so the toggle can register the app normally.

Install Notes

  • Keep AppWink.app installed at /Applications/AppWink.app or ~/Applications/AppWink.app if you want the cleanest signed-bundle and login-item behavior.
  • Users already on the live 0.1.14 (15) Sparkle build should upgrade in place to pick up the launch-at-login repair.

Known Limitations

  • Fully automated verification of AppWink's Carbon global hotkeys is still unreliable in this environment, so hotkey closure still depends on real manual smoke.
  • GitHub Actions tag-release automation still depends on repository secrets that were previously missing, so this release may still rely on the documented maintainer-local notarization and upload flow.

Verification Scope

  • swift build --disable-sandbox and swift test --disable-sandbox are rerun as the release baseline on this release line.
  • Signed local packaging, notarization, and DMG validation are rerun for 0.1.15 (16) before publication.
  • On a signed installed bundle, Toolbox > Other > Launch at Login now enables, toggles successfully, and adds/removes AppWink under macOS Login Items.
